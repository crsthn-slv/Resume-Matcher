## Interaction Contract

### Shared Badge Trigger

- The current visible status badge becomes the only trigger for status editing.
- Clicking the badge opens a compact dropdown anchored to the badge itself.
- Dashboard and resume viewer must use the same visual trigger and the same dropdown behavior.
- The dropdown closes immediately after selection.

### Dashboard Cards

- Every tailored-resume card shows a status badge in the existing application-signal area.
- Cards with an existing application open the dropdown and save the chosen status immediately.
- Cards without an existing application still open the same dropdown.
- For cards without an application, selecting a status first attempts quick-create using inferred `company`, `role`, and linked job/resume context.
- If inference is insufficient, the UI falls back to the normal create-application flow with prefill rather than showing a dead-end error at the card level.

### Resume Viewer

- The dedicated “status change” block is removed from the primary application panel.
- The main status badge in the application details grid becomes the inline editing trigger.
- Status history remains visible below and continues to reflect successful persisted changes only.

### Saving Feedback

- Selection is optimistic: the badge text and variant switch immediately on selection.
- During save, the badge keeps its new value and shows a compact spinner inside the badge.
- The spinner must not change the row height or push surrounding content noticeably.
- While a save is pending, the same badge cannot be reopened.

### Failure Feedback

- On failure, the badge reverts to its previous value and visual style.
- The error is communicated globally using a transient app-level notification pattern, not a permanent inline error panel near the badge.
- The notification copy is short and action-oriented.

## Component Contract

### Dashboard Badge

- Reuse `CardBadge` styling as the base visual language.
- Add interactive affordance with minimal change:
  - pointer cursor
  - retained hard border and shadow
  - no rounded-pill styling
  - no oversized select input chrome
- The dropdown width should align to the badge width at minimum and may expand modestly for readable option labels.

### Inline Dropdown

- Dropdown surface is white with black border and hard shadow, matching existing Swiss-style controls.
- Options use mono labels with compact vertical rhythm.
- The currently selected status row uses the same accent family already mapped for status meaning.
- No long helper description block is shown in the inline dropdown trigger; labels should stay terse.

### Global Notification

- Prefer the app’s existing custom notification/dialog styling vocabulary over introducing a third-party toast library.
- Notification should feel lightweight and non-blocking.
- Copy should avoid stack-trace detail and focus on retryable user language.
