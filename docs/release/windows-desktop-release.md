# Windows Desktop Release

## Version source

The Windows installer version comes from `apps/desktop/package.json`. Bump that version before creating a release tag.

## Canonical CI release

GitHub Actions is the official release path for the Windows desktop build.

1. Update `apps/desktop/package.json` to the intended release version.
2. Push a tag in the format `v*.*.*`, for example `v1.1.0`, or trigger the `Windows Desktop Release` workflow manually.
3. Let `.github/workflows/windows-desktop-release.yml` build the runtime bundle and package the installer.
4. Collect the uploaded artifact from `dist/desktop-installer`.

The workflow runs on `windows-latest` and calls `npm run desktop:release:win`.

## Local fallback

Use a local Windows machine only when GitHub Actions is unavailable.

From the repository root:

```bash
npm run desktop:release:win
```

For an unpacked directory build instead of the final installer:

```bash
npm run desktop:release:win:dir
```

Local Windows execution is fallback only. The official release still comes from GitHub Actions in the main repository.

## Artifacts

- Installer output directory: `dist/desktop-installer`
- Installer naming pattern: `ResumeMatcher-Setup-${version}.exe`
- Unpacked directory fallback: generated by `npm run desktop:release:win:dir`

v1.1 does not add auto-update infrastructure, update feeds, or a separate distribution fork.
